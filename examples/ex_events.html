Céu interacts with the environment by awaiting external input events, such as MY_EVT in the example.

The await statement accounts for the reactive nature of Céu.

An input event in Céu represents a correspondent event in the underlying platform. The availability of external events depends on the platform in use.

As an example, in an Arduino binding for Céu, a change in an input pin on the microcontroller triggers a correspondent input event in Céu.

In higher-level platforms (e.g. operating systems), more abstract events might be available, such as keyboard presses, or the arrival of network packets.
